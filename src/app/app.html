Filter Tips

  • By default (regex icon off), the global filter and text columns use expression syntax: build conditions with && (AND), || (OR), unary ! (NOT), and parentheses. Operands can include spaces (e.g. green land && blue sky); only the start and end of each operand are trimmed, and spaces around operators are optional. Example: (red || blue) && orange.
  • Without quoted operands, you cannot search for the literal substrings &&, ||, or parentheses inside a single term (a future improvement could add quoting).
  • The case-sensitive and whole-word icon toggles (same icons as in VS Code) apply whether you use expression syntax or regex: expression operands respect them, and bare regex patterns use them for default flags and word boundaries (explicit /pattern/flags keeps your flags; whole-word still wraps the pattern). Active toggles use the same accent border as the regex icon button when it is on.
  • Turn on the regex icon (next to case / whole word) to treat the whole field as one pattern. That button uses the same accent border as the Export / Import config toolbar when active (global and per text column).
  • List-style column filters still use comma-separated tokens with their own OR/AND row (unchanged).
  • Global scope defaults to visible columns; use the scope dropdown to search all columns instead.
  • Invalid expression syntax matches no rows until you fix the filter (unbalanced parentheses, stray operators, etc.).
  • Global highlights are blue; column-specific highlights are green.
  • Click chips to remove active filters quickly.
