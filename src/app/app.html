Filter Tips

  • In Expression mode (global filter and text columns), build conditions with && (AND), || (OR), unary ! (NOT), and parentheses. Operands can include spaces (e.g. green land && blue sky); only the start and end of each operand are trimmed, and spaces around operators are optional. Example: (red || blue) && orange.
  • Without quoted operands, you cannot search for the literal substrings &&, ||, or parentheses inside a single term (a future improvement could add quoting).
  • Between Expression and Regex, the case-sensitive and whole-word icon toggles (same icons as in VS Code) apply to both modes: expression operands respect them, and bare regex patterns use them for default flags and word boundaries (explicit /pattern/flags keeps your flags; whole-word still wraps the pattern). Active toggles use the same accent border as the mode buttons.
  • Regex mode treats the whole field as one pattern. The active mode toggle (Expression or Regex) uses the same border color as the Export / Import config toolbar buttons (global and per text column).
  • List-style column filters still use comma-separated tokens with their own OR/AND row (unchanged).
  • Global scope can target either all columns or visible columns.
  • Invalid expression syntax matches no rows until you fix the filter (unbalanced parentheses, stray operators, etc.).
  • Global highlights are blue; column-specific highlights are green.
  • Click chips to remove active filters quickly.
